# QNAP

本文档介绍如何在QNAP存储上实现基本的管理及增加各类应用。

## 技巧：共享文件夹实现高级权限

问题：在QNAP存储的共享文件夹中创建一个共享文件夹XIN，在该文件夹中创建两个目录XIN-客户合同和XIN-采购合同，项目管理部可以对XIN-客户合同有读写权限，对XIN-采购合同有只读权限，商务部反过来如此。升级uv

### 1. 创建主共享文件夹

- 登录 QNAP Web 管理界面。
- 进入 **控制面板 → 权限 → 共享文件夹**。
- 点击 **创建 → 共享文件夹**，名称填写XIN
- 权限先设为“仅管理员可访问”，稍后再细分。

### 2. 建立两个子目录

在 File Station中创建两个子目录，XIN-客户合同和XIN-采购合同

### 3. 建立两个用户组

进入 **控制面板 → 权限 → 用户 → 用户组**

- 组 1：项目管理部
- 组 2：商务部

如果已有，可跳过。

### 4. 设置目录权限

**启用高级文件夹权限（ACL）**

打开 QNAP 控制面板，控制面板 → 权限 → 共享文件夹

选中主文件夹 XIN → 点击 “编辑共享文件夹权限”

在打开的窗口中，找到下方的选项 “启用高级文件夹权限”（或“高级权限设置”），勾选启用后点击「应用」

返回 File Station → 右键文件夹 → 选择 “属性” → “权限”，现在会出现一个更丰富的权限表格界面。

**设置目录权限**

在File Station → 右键文件夹 → 选择 “属性” → “权限”中，点击 “添加”，选择 用户组（例如“项目管理部”“商务部”），分别设置 读写 / 只读 / 禁止访问：

| 文件夹       | 项目管理部 | 商务部 |
| ------------ | ---------- | ------ |
| XIN-客户合同 | 读写       | 只读   |
| XIN-采购合同 | 只读       | 读写   |

如果是新建的目录结构，建议勾选 “应用并替换此文件夹、其文件及子文件夹的权限”，强制替换掉原有权限。

选项“应用并更改此文件夹、其文件及子文件夹权限”，将在在保留已有 ACL 权限结构的前提下，将当前设置“追加”或“更新”到现有文件和子目录。
